## Task 2: Add `plan.subs.drop_unsupported` to the `hevc-normalize` flow

**Files:**

```yaml
  # Drop data streams (incompatible with most containers anyway).
  - id: plan-drop-data
    use: plan.streams.drop_data

  # If the video is NOT already hevc, mark it for re-encode.
  # (When already hevc, the plan keeps video as `copy` and we only do the

```

Replace with:

```yaml
  # Drop data streams (incompatible with most containers anyway).
  - id: plan-drop-data
    use: plan.streams.drop_data

  # Drop subtitle streams whose codec doesn't mux into the planned container
  # (e.g. mov_text from MP4 sources can't go into MKV). Without this, ffmpeg
  # bails at MKV muxer init with "Function not implemented" before any
  # encoding starts, and the broken near-empty tmp file confuses verify.
  - id: plan-drop-unsupported-subs
    use: plan.subs.drop_unsupported

  # If the video is NOT already hevc, mark it for re-encode.
  # (When already hevc, the plan keeps video as `copy` and we only do the
  # audio + container work in plan.execute.)
  - id: codec-gate
```

Placement is *after* `drop_data` / `drop_cover_art` (the "cull obviously-unwanted streams" prefix) and *before* `codec-gate` (so the check applies whether or not we re-encode video — a copy-only pass with mov_text → MKV would fail just the same).

## Task 3: Capture & check CPU fallback exit status in `plan.execute`

**Files:**

- [ ] **Step 1: Apply the edit**

In `crates/transcoderr/src/steps/plan_execute.rs`, find this block (the `Ok(status) => { ... }` arm of the `match result`):

```rust
            Ok(status) => {
                // Ran but exited non-zero. CPU fallback if HW was used.
                if acquired_key.is_some() && plan.video.hw_fallback_cpu {
                    on_progress(StepProgress::Marker {
                        kind: "hw_runtime_failure".into(),
                        payload: json!({ "device": acquired_key }),
                    });
                    let _ = std::fs::remove_file(&dest);
                    let cpu_cmd = build_command(&src, &dest, &plan, &probe, None)?;
                    let _ = crate::ffmpeg::run_with_live_events(
                        cpu_cmd,

                        ctx.cancel.as_ref(),

                            FfmpegEvent::Pct(p) => on_progress(StepProgress::Pct(p)),
                            FfmpegEvent::Line(l) => {
                                on_progress(StepProgress::Log(format!("ffmpeg: {l}")))
                            }
                        },
                    )
                    .await?;
                    staging::record_output(
                        ctx,
                        &dest,
                        json!({ "hw": null, "fallback_from": acquired_key }),
                    );
                    Ok(())
                } else {
                    anyhow::bail!("plan.execute: ffmpeg exited {:?}", status.code())
                }
            }
```

Replace with:

```rust
            Ok(status) => {
                // Ran but exited non-zero. CPU fallback if HW was used.
                if acquired_key.is_some() && plan.video.hw_fallback_cpu {
                    on_progress(StepProgress::Marker {
                        kind: "hw_runtime_failure".into(),
                        payload: json!({ "device": acquired_key }),
                    });
                    let _ = std::fs::remove_file(&dest);
                    let cpu_cmd = build_command(&src, &dest, &plan, &probe, None)?;
                    let cpu_status = crate::ffmpeg::run_with_live_events(
                        cpu_cmd,
                        duration_sec,
                        ctx.cancel.as_ref(),
                        |ev| match ev {
                            FfmpegEvent::Pct(p) => on_progress(StepProgress::Pct(p)),
                            FfmpegEvent::Line(l) => {
                                on_progress(StepProgress::Log(format!("ffmpeg: {l}")))
                            }
                        },
                    )
                    .await?;
                    if !cpu_status.success() {
                        anyhow::bail!(
                            "plan.execute: cpu fallback also failed (hw exited {:?}, cpu exited {:?})",
                            status.code(),
                            cpu_status.code()
                        );
                    }
                    staging::record_output(
                        ctx,
                        &dest,
                        json!({ "hw": null, "fallback_from": acquired_key }),
                    );
                    Ok(())
                } else {
                    anyhow::bail!("plan.execute: ffmpeg exited {:?}", status.code())
                }
            }
```

The two-line diff:
- `let _ = crate::ffmpeg::run_with_live_events(...).await?;` → `let cpu_status = crate::ffmpeg::run_with_live_events(...).await?;`
- New `if !cpu_status.success() { bail! }` block before `record_output` is called.

The bail message includes BOTH exit codes — when an operator sees `hw exited 234, cpu exited 234`, identical codes on both attempts immediately tell them the failure is upstream of the encoder.
